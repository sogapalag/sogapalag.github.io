about gnome and gpu crash

Posted on 2019-08-28 Edited on 2019-11-22

hardware info

ryzen 2600, b450m-ds3h, gtx-970

os info

ubuntu 18.04.3, gnome3.28.2, kernal5.0.0-25-generic

Problem Description

Since I bulid the brand new PC except my old gpu. I suffered contantly, frequently freeze/crash.

some cases

  • chrome showed “aw,snap” or apps starting flashing, and some apps doesn’t respond, a few seconds screen freezing, then mouse doesn’t respond.
  • screen back to login window.
  • sudden freezing
  • chrome video(bilibili) crash possibility is higher than normal website.

At first I replace nouveau with nvidia-390, after then PC works fine about 3 weeks, no crash. But these days the problem came up again. After crash check logs

org.gnome.Shell.desktop[1608]: [3114:3145:0828/173046.809153:ERROR:ssl_client_socket_impl.cc(943)] handshake failed; returned -1, SSL error code 1, net_error -…
NVRM: Xid 69/12/32 error

and many other error logs. most correlated to gnome and gpu. sometimes cpu soft lockup.

what I tried

  • test firefox (not work)
  • disable amd c6 state. (not work)
  • change gnome extensions (not work)
  • set gnome back to default (not work)
  • use different drivers(nvidia-430, 418, 390, by apt, by mannully) (not work)
  • uninstall all drivers, use default llvmpipe (LLVM 8.0, 128 bits). (work, so the problem is related to gpu)
  • use 1660 ti instead of 970, (work, 144hz also work. but didn’t test much time. problem occur. hdmi-120 fine.)
  • 970 on old board with i7. (work, but in vim freezing 1s every 5 mins. old hardware issue?)
  • weird situation, 1660ti with amd, windows 10 pro(1803/1903), driver(419/430//436) dp(119/120/144), blue screen. now, 1903, 436, hdmi(120), no problem at least one day, no problem yet. so dp wire or port issue?
  • win10+1660ti+2600, use new dp wire, 144hz, stable more than 3 days. (so all problem is about the wire??)

although change gnome extensions doesn’t solve the problem, but I do feel the possibility of crash is lower.(or I’m wrong) If I change themes in tweaks, back&on, easily crash.

Some thoughts and next step

crash is annoying since you will get many error logs, and cannot find what the exact cause, since it might be hareware, software or even both. For my problem, I suspected cpu, power supply, gnome, X server, gpu broken, drivers. but I still no clue. Well after tried some test, now I believe it should be gpu hardware or drivers issue. but I don’t know why after 3 weeks stable time, the problem show up again, I don’t remember I did anything special.

next, I might try to replace with my old boadrd and old i7-4790k. test the compatibility. (since it works fine a fewer years ago with old setting)

Updated info

2019-10-20, ubuntu,amd,1660ti. 60hdmi. start crash again, about 10 times in two hours. don’t know why.

2019-10-20 to 10-31, after change Memory-bar from slot 3 to slot 1. stable till 10-31. then cannot start-up, BIOS show ROM not loaded. then change to slot 3, start-up but sometimes freeze. now using slot 2 stable.

2019-11-20, crash again, after try different slots, finally slot 3 works.
